# MintPlayer.Spark.Cron

Cron-scheduled background jobs for [MintPlayer.Spark](https://github.com/MintPlayer/MintPlayer.Spark).

Define a job by implementing `ISparkCronJob` and declaring its schedule with a `static abstract`
member — the schedule lives with the job, not at the registration call site. The scheduler runs
each job on its cron schedule, and is safe to run on multiple nodes: a RavenDB compare-exchange
claim ensures every occurrence fires exactly once across the cluster.

## Defining a job

```csharp
using MintPlayer.Spark.Cron;
using MintPlayer.SourceGenerators.Attributes;
using Raven.Client.Documents.Session;

public partial class NightlyCleanup : ISparkCronJob
{
    public static string CronSchedule => "0 0 * * *"; // 00:00 UTC daily

    [Inject] private readonly IAsyncDocumentSession session;
    [Inject] private readonly ILogger<NightlyCleanup> logger;

    public async Task RunAsync(CancellationToken cancellationToken)
    {
        logger.LogInformation("Cleaning up...");
        // ... use session ...
        await session.SaveChangesAsync(cancellationToken);
    }
}
```

The job is resolved from a fresh DI scope for every run, so `[Inject]` dependencies behave like
any scoped service.

### Schedule

`CronSchedule` is an [NCrontab](https://github.com/atifaziz/NCrontab) expression, always interpreted
in **UTC**:

- Five fields (`m h dom mon dow`) — minute precision.
- Six fields (`s m h dom mon dow`) — second precision (auto-detected by field count).

For a job that should run at a local wall-clock time, write the UTC equivalent. (UTC schedules do
not auto-shift across daylight-saving transitions.)

### Concurrency

By default a still-running occurrence suppresses the next one on the same node. Override
`AllowConcurrentRuns` to let occurrences overlap:

```csharp
public static bool AllowConcurrentRuns => true;
```

## Registration

```csharp
builder.Services.AddSpark(builder.Configuration, spark =>
{
    spark.UseContext<MyContext>();

    // Explicit:
    spark.AddCron(cron => cron.AddJob<NightlyCleanup>());

    // ...or auto-discovered via the source generator:
    spark.AddCronJobs();
});
```

`spark.AddCronJobs()` is generated by `MintPlayer.Spark.SourceGenerators`: it discovers every
`ISparkCronJob` in the project and registers it. Both entry points register the scheduler exactly
once, so they compose freely.
