# PRD: MintPlayer.Spark.AllFeatures

## Problem Statement

Setting up a full-featured Spark application currently requires:
1. Referencing 5+ NuGet packages individually (Core, Authorization, Messaging, Replication, SourceGenerators)
2. Calling numerous builder methods inside `AddSpark()` — `UseContext<T>()`, `AddActions()`, `AddAuthorization()`, `AddAuthentication<T>()`, `AddMessaging()`, `AddRecipients()`, `AddReplication()`, etc.
3. Calling `SynchronizeSparkModelsIfRequested<TContext>(args)` as a separate top-level call outside the Spark pipeline

Developers who want "everything" should reference a single package and write:

```csharp
builder.Services.AddSparkFull(builder.Configuration);
app.UseSparkFull(args);
endpoints.MapSparkFull();
```

## Goals

1. **Single NuGet package** — `MintPlayer.Spark.AllFeatures` brings in all Spark libraries + source generators
2. **Three-method Program.cs** — `AddSparkFull` / `UseSparkFull` / `MapSparkFull`
3. **Source-generated** — the three methods are generated at compile time with hardcoded types discovered from the consuming project
4. **`SynchronizeSparkModelsIfRequested` absorbed** into `UseSparkFull`
5. **Typed options** — `SparkFullOptions` exposes per-feature configuration (Authorization, Identity, Messaging, Replication)

## Non-Goals

- Replacing the existing granular API (remains available for apps that need a subset)
- Changing how individual modules work internally
- Adding new functionality to any existing module

## Design

### Architecture

Two .csproj projects that produce **one NuGet package** (same pattern as Authorization with its `.props`/`.targets`):

| Project | Target | Role |
|---|---|---|
| `MintPlayer.Spark.AllFeatures` | net10.0 (Web SDK) | Shell: references all Spark packages (become NuGet deps), houses `SparkFullOptions`, packs analyzers, ships `.targets` |
| `MintPlayer.Spark.AllFeatures.SourceGenerators` | netstandard2.0 | Source generator: emits `AddSparkFull` / `UseSparkFull` / `MapSparkFull` |

### NuGet Package Layout

When packed, `MintPlayer.Spark.AllFeatures.nupkg` contains:

```
lib/
  net10.0/
    MintPlayer.Spark.AllFeatures.dll        (SparkFullOptions + transitive deps)
analyzers/
  dotnet/
    cs/
      MintPlayer.Spark.AllFeatures.SourceGenerators.dll   (the new generator)
      MintPlayer.Spark.SourceGenerators.dll               (existing generator, bundled)
buildTransitive/
  MintPlayer.Spark.AllFeatures.targets      (for NuGet: wires up analyzers)
```

NuGet dependencies declared:
- `MintPlayer.Spark`
- `MintPlayer.Spark.Authorization`
- `MintPlayer.Spark.Messaging`
- `MintPlayer.Spark.Replication`
- (NOT `MintPlayer.Spark.SourceGenerators` — bundled as analyzer instead)

**NuGet consumer** — single reference:
```xml
<PackageReference Include="MintPlayer.Spark.AllFeatures" Version="10.0.0-preview.27" />
```

### Local Development (ProjectReference)

For local development in the solution, follows the same pattern as `MintPlayer.Spark.Authorization` (`.targets` import):

```xml
<Import Project="..\..\..\MintPlayer.Spark.AllFeatures\Targets\spark-allfeatures.targets" />
<ProjectReference Include="..\..\..\MintPlayer.Spark.AllFeatures\MintPlayer.Spark.AllFeatures.csproj" />
```

The `.targets` file adds the analyzer `<ProjectReference>` entries that can't flow transitively through project references.

### Source Generator: Type Discovery

The generator scans the consuming project's compilation to detect:

| What | How | Drives |
|---|---|---|
| `SparkContext` subclass | Non-abstract class inheriting `SparkContext` | `UseContext<T>()`, `SynchronizeSparkModelsIfRequested<T>()` |
| `SparkUser` subclass | Non-abstract class inheriting `SparkUser` | `AddAuthentication<T>()` (falls back to `SparkUser` if no subclass) |
| Actions classes exist | Any non-abstract class inheriting `DefaultPersistentObjectActions<T>` | Conditionally emit `AddActions()` call |
| Custom actions exist | Any non-abstract class implementing `ICustomAction` | Conditionally emit `AddCustomActions()` call |
| Recipients exist | Any non-abstract class implementing `IRecipient<T>` | Conditionally emit `AddRecipients()` call |
| Authorization referenced | `SparkBuilderAuthorizationExtensions` resolvable | Conditionally emit `AddAuthorization()` call |
| Messaging referenced | `SparkBuilderMessagingExtensions` resolvable | Conditionally emit `AddMessaging()` call |
| Replication referenced | `SparkBuilderReplicationExtensions` resolvable | Conditionally emit `AddReplication()` call (guarded by `options.Replication != null`) |

The generator emits calls to methods already generated by `MintPlayer.Spark.SourceGenerators` (`AddActions()`, `AddRecipients()`, `AddCustomActions()`). Since both generators emit into the same compilation, the C# compiler resolves all calls after all generators have run.

### SparkFullOptions

The shell project (`MintPlayer.Spark.AllFeatures`) contains a typed options class that exposes per-feature configuration:

```csharp
public class SparkFullOptions
{
    public Action<AuthorizationOptions>? Authorization { get; set; }
    public Action<IdentityOptions>? Identity { get; set; }
    public Action<IdentityBuilder>? IdentityProviders { get; set; }
    public Action<SparkMessagingOptions>? Messaging { get; set; }
    public Action<SparkReplicationOptions>? Replication { get; set; }
}
```

Each property maps to the corresponding module's configuration callback. When null, the module uses its defaults. Replication is only enabled when `Replication` is explicitly set (it requires mandatory configuration like `ModuleName` and `ModuleUrl`).

### Generated Methods

#### `AddSparkFull`

```csharp
// Generated — types hardcoded from discovery
internal static class SparkFullBuilderExtensions
{
    public static IServiceCollection AddSparkFull(
        this IServiceCollection services,
        IConfiguration configuration,
        Action<SparkFullOptions>? configure = null)
    {
        var options = new SparkFullOptions();
        configure?.Invoke(options);

        services.AddSpark(configuration, spark =>
        {
            spark.UseContext<FleetContext>();       // discovered SparkContext subclass
            spark.AddActions();                     // if actions classes found
            spark.AddCustomActions();               // if custom action classes found
            spark.AddRecipients();                  // if recipient classes found
            spark.AddAuthorization(options.Authorization);   // if Authorization package referenced
            spark.AddAuthentication<SparkUser>(              // if SparkUser found
                options.Identity, options.IdentityProviders);
            spark.AddMessaging(options.Messaging);           // if Messaging package referenced
            if (options.Replication != null)                  // if Replication package referenced
                spark.AddReplication(options.Replication);
        });
        return services;
    }
}
```

#### `UseSparkFull`

```csharp
public static IApplicationBuilder UseSparkFull(
    this IApplicationBuilder app,
    string[] args)
{
    app.UseSpark(spark =>                                      // uses the new UseSpark overload
    {
        spark.SynchronizeModelsIfRequested<FleetContext>(args); // discovered context type
    });
    return app;
}
```

The generated code uses the `UseSpark(Action<UseSparkOptions>)` overload added to the core `MintPlayer.Spark` package. This overload is also available for granular setups:

```csharp
app.UseSpark(o => o.SynchronizeModelsIfRequested<MyContext>(args));
```

#### `MapSparkFull`

```csharp
public static IEndpointRouteBuilder MapSparkFull(
    this IEndpointRouteBuilder endpoints)
{
    endpoints.MapSpark();
    return endpoints;
}
```

### Complete Examples

**Simplest case** — no per-feature configuration needed:

```csharp
var builder = WebApplication.CreateBuilder(args);

builder.Services.AddSparkFull(builder.Configuration);

var app = builder.Build();

app.UseRouting();
app.UseSparkFull(args);
app.UseEndpoints(endpoints =>
{
    endpoints.MapControllers();
    endpoints.MapSparkFull();
});

app.Run();
```

**With replication and custom identity options:**

```csharp
builder.Services.AddSparkFull(builder.Configuration, options =>
{
    options.Identity = id =>
    {
        id.Password.RequiredLength = 10;
    };
    options.Replication = opt =>
    {
        opt.ModuleName = "Fleet";
        opt.ModuleUrl = "https://localhost:5003";
        opt.SparkModulesUrls = ["http://localhost:8080"];
        opt.AssembliesToScan = [typeof(Fleet.Replicated.Person).Assembly];
    };
});
```

### Generator Implementation

Follows the established pattern from `MintPlayer.Spark.SourceGenerators`:

| Component | File | Purpose |
|---|---|---|
| `SparkFullDiscoveredType` | `Models/SparkFullDiscoveredType.cs` | Carries discovered type info (kind + fully qualified name) |
| `SparkFullFeatureFlags` | `Models/SparkFullFeatureFlags.cs` | Compilation-level package availability flags |
| `SparkFullGenerator` | `Generators/SparkFullGenerator.cs` | `IncrementalGenerator` — discovers types, combines providers |
| `SparkFullProducer` | `Generators/SparkFullGenerator.Producer.cs` | `Producer` — emits the three extension methods |

**Detection logic** (in `SparkFullGenerator`):
1. **SyntaxProvider**: Scans all classes with base types for:
   - `SparkContext` subclass → context type name
   - `SparkUser` subclass → user type name
   - `DefaultPersistentObjectActions<T>` subclass → `hasActions`
   - `ICustomAction` implementor → `hasCustomActions`
   - `IRecipient<T>` implementor → `hasRecipients`
2. **CompilationProvider**: Checks metadata names for package availability:
   - `SparkContext` → `hasSpark`
   - `SparkUser` → `hasSparkUser` (for fallback when no subclass found)
   - `SparkBuilderAuthorizationExtensions` → `hasAuthorization`
   - `SparkBuilderMessagingExtensions` → `hasMessaging`
   - `SparkBuilderReplicationExtensions` → `hasReplication`

If no `SparkContext` subclass is found, nothing is generated.
